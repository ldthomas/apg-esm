APG — Grammar Trace Visualizer — User Guide

Overview

The APG Grammar Trace Visualizer renders the grammar-level trace produced by an APG parser in a step-by-step, animated format. Each step corresponds to one node traversal event recorded by the traceSabnf object during a parse, displayed in the context of the original SABNF grammar text.

The page's main display, the left panel, shows the parse tree as node operators for each step, indented to show the tree depth of the node, and followed by the line:offset of grammar text controlling the parser's operation. The right panel displays the grammar text, with highlighting to show the parser's current location of operation, current node information and the parsing progress.

The title bar has all of the controls for loading the trace into the main panel and various methods for stepping through the tree. The footer contains a legend for the color coding of the node states.

Loading Trace Data

The Grammar Trace Visualizer provides a textarea to enter a parse tree (output from traceSabnf.display()). There are three main ways to access this.

  1. The primary way is from the Parser Generator page (./docs/index.html). After generating a grammar object from an SABNF grammar and parsing a string with the Trace Grammar Text box checked, the Trace Grammar Text tab will have a link with instructions for accessing the Grammar Trace Visualizer
  2. Once the Visualizer page has been displayed, clicking the Load New Trace button in the header will display the input textarea for a new parse tree.
  3. The Visualizer page can also be displayed directly from its file URL (file://<path to project>/docs/grammar-visualizer.html). The input textarea will automatically be displayed.

Navigating the Trace

There are several options for stepping through the parse tree.

  • ▶ Play — starts an animation that advances through the tree automatically. The rightmost slider controls playback speed.
  • ◀ Prev — moves one step back. The ↑ and ← arrow keys are shortcuts for this button.
  • Next ▶ — moves one step forward. The ↓ and → arrow keys are shortcuts for this button.
  • Position slider (to the right of the Next button) — drag to jump directly to any position in the tree.
  • Row click — clicking any row in the parse tree advances the display to that step.

Trace Output Format

Once a parse tree has been loaded the tree displays in the left panel, one line for each step the parser has taken. Each line has five or six parts.

  • The node's tree depth, indicated by the indentation.
  • The parser's state.
    • ↓ Entering — the parser is descending into this node.
    • M Match — the node matched one or more characters.
    • E Empty — the node matched zero characters (valid empty match).
    • N No-match — the node failed to match any characters.
  • The node operator†. For complete node operator details see the SABNF Grammar Reference.

    • † UDT not available here because the generator page is incapable of providing for the required user-written callback function.

  • ↑ or ↓ indicating the upward or downward direction of tree traversal
  • line:offset indication of the grammar text line and character offset of the parser's position in the grammar text
  • If the state is M Match, the matched characters. Phrases longer than MAX_PHRASE characters are truncated and end with ....

Grammar Text Panel

At each step, the grammar text panel highlights the single character at the parser's current position. The background color of that character indicates the parser's state:

  • c  Entering — blue; the parser is descending into the node at this position.
  • c  Match — green; the node matched one or more characters starting here.
  • c  Empty — yellow; the node matched zero characters (valid empty match) at this position.
  • c  No-match — red; the node failed to match at this position.

ALT and CAT operators are present in the grammar but have no explicit character representation in the flat grammar text — alternation is implied by the / separator and concatenation by juxtaposition. Because there is no single character to point to, their highlight appears at the first character of the grammar line on which they occur.

CURRENT step

The current step panel below the input string gives a self-explanatory summary of the current step.

PROGRESS

The progress panel below the current step gives a self-exlanatory summary of the number of steps and states up to and including the current node.
