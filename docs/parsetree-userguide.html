APG — Parse Tree Visualizer — User Guide

Overview

The APG Parse Tree Visualizer renders the parse tree produced by an APG parser in a step-by-step, animated format. Each step corresponds to one node traversal event recorded by the trace object during a parse.

The page's main display, the left panel, shows the parse tree as node operators for each step, indented to show the tree depth of the node, and followed by the string being parsed. The right panel displays the input string, with highlighting to show the matched characters, current node information and the parsing progress.

The title bar has all of the controls for loading the trace into the main panel and various methods for stepping through the tree. The footer contains a legend for the color coding of the node states.

Loading Trace Data

The Parse Tree Visualizer provides a textarea to enter a parse tree (output from trace.display()). There are three main ways to access this.

  1. The primary way is from the Parser Generator page (./docs/index.html). After generating a grammar object from an SABNF grammar and parsing a string with the Trace Parse Tree box checked, the Trace Parse Tree tab will have a link with instructions for accessing the Parse Tree Visualizer
  2. Once the Visualizer page has been displayed, clicking the Load New Trace button in the header will display the input textarea for a new parse tree.
  3. The Visualizer page can also be displayed directly from its file URL (file://<path to project>/docs/parsetree-visualizer.html). The input textarea will automatically be displayed.

Navigating the Parse Tree

There are several options for stepping through the parse tree.

  • ▶ Play — starts an animation that advances through the tree automatically. The rightmost slider controls playback speed.
  • ◀ Prev — moves one step back. The ↑ and ← arrow keys are shortcuts for this button.
  • Next ▶ — moves one step forward. The ↓ and → arrow keys are shortcuts for this button.
  • Position slider (to the right of the Next button) — drag to jump directly to any position in the tree.
  • Row click — clicking any row in the parse tree advances the display to that step.

Trace Output Format

Once a parse tree has been loaded the tree displays in the left panel, one line for each step the parser has taken. Each line has four parts.

  • The node's tree depth, indicated by the indentation.
  • The parser's state.
    • ↓ Entering — the parser is descending into this node.
    • M Match — the node matched one or more characters.
    • E Empty — the node matched zero characters (valid empty match).
    • N No-match — the node failed to match any characters.
  • The node operator†. For complete node operator details see the SABNF Grammar Reference.

    • † UDT not available here because the generator page is incapable of providing for the required user-written callback function.

  • The string being parsed if entering state, the string matched on a successful node operation: ε if emtpy, or ✗ if no match is found.

INPUT STRING

The input string is displayed in the upper portion of the right panel. Highlighting indicates the parser's progress. When it is in an enter state, the first character of the string the parser is attempting to match will be highlighted in light blue. Each time a character or phrase is matched, those characters are highlighted in green. On failure, or the no match state, the failing character is highlighted in red. In this way the viewer can follow the parser's path through the input string.

CURRENT NODE

The current node panel below the input string gives a mostly self-explanatory summary of the currently highlighted node. Note that Look-ahead yes means that the node is on a branch whose parent node is an AND or NOT operator.

PROGRESS

The progress panel below the current node gives a self-exlanatory summary of the number of steps and states up to and including the current node.
